# Azure 앱 등록 설정 지침 — 동글랜드 런처 v3

로그인 방식:
- **기본**: 앱 내부 로그인 창(임베디드 웹뷰) — authorization code flow
- **대체**: 인증 코드(device code) — 로그인 버튼 아래 "인증 코드로 로그인"

임베디드/브라우저 방식은 로컬 콜백을 쓰므로 **리디렉션 URI 등록이 필요**하다.
device code 는 리디렉션 URI 가 필요 없다.

| 방식 | 필요한 리디렉션 URI |
|------|---------------------|
| 임베디드 웹뷰 / 브라우저 폴백 | `http://localhost` |
| 인증 코드(device code) | (불필요) |

(WAM 네이티브 계정창은 서드파티 클라이언트에서 사용 불가 → 채택 안 함)

## 1단계 — 앱 등록 열기

1. https://portal.azure.com 접속
2. 상단 검색창에 **"앱 등록"** (또는 App registrations) 입력 → 선택
3. 목록에서 해당 앱 클릭
   - 안 보이면 상단 탭을 **"모든 애플리케이션"**으로 변경
   - **개요(Overview)** 에서 애플리케이션(클라이언트) ID 가 위 값과 같은지 확인

---

## 2단계 — 지원 계정 유형 확인 (중요)

**개요** 화면 → "지원되는 계정 유형" 항목이 아래여야 한다:

> **개인 Microsoft 계정만 해당** (Personal Microsoft accounts only)

또는 "모든 조직 디렉터리의 계정 및 개인 Microsoft 계정"도 가능.
**"내 조직 디렉터리의 계정만"** 으로 되어 있으면 다른 사람이 로그인 못 한다.

바꾸려면: **인증(Authentication)** → 맨 아래 "지원되는 계정 유형" 에서 변경 → 저장.

---

## 3단계 — 플랫폼 추가 (리디렉션 URI 등록)

**인증(Authentication)** 블레이드로 이동.

1. **"플랫폼 추가"** 클릭
2. **"모바일 및 데스크톱 애플리케이션"** 선택
   (⚠️ "웹" 이나 "SPA" 아님 — 반드시 모바일 및 데스크톱)
3. **"사용자 지정 리디렉션 URI"** 입력란에 아래를 그대로 입력 (포트 번호 없이)
   ```
   http://localhost
   ```
4. **"구성"** 클릭

> Azure 는 `localhost` 리디렉션에서 **포트를 무시**하므로,
> 런처가 여는 임의 포트(`http://localhost:52341/` 등)가 전부 이 등록과 매칭된다.
> 그래서 포트를 적으면 안 되고, 여러 포트를 등록할 필요도 없다.

> ⚠️ `http://127.0.0.1` 은 포털 입력란으로 추가할 수 없다(매니페스트 직접 편집 필요).
> 그래서 코드도 `localhost` 를 쓰도록 맞춰 두었다.

---

## 4단계 — 퍼블릭 클라이언트 흐름 허용

**인증(Authentication)** 블레이드 맨 아래:

> **"고급 설정" → "퍼블릭 클라이언트 흐름 허용"** → **예(Yes)** 로 설정 → 저장

이게 꺼져 있으면 client secret 없이 로그인이 안 된다.

---

## 5단계 — 최종 확인

**인증** 블레이드에서 아래 상태여야 한다:

```
플랫폼 구성
└─ 모바일 및 데스크톱 애플리케이션
   └─ http://localhost

고급 설정
└─ 퍼블릭 클라이언트 흐름 허용: 예
```

**개요** 블레이드:
```
지원되는 계정 유형: 개인 Microsoft 계정만 해당
```

## 문제 해결

### `AADSTS50011: redirect URI ... does not match`
`http://localhost` 미등록. 에러 메시지에 실제 요청된 URI(`http://localhost:12345/`)가
찍히지만, 포털에는 **포트 없이 `http://localhost`** 만 등록하면 된다(포트는 무시됨).

### 앱 내부 로그인 창 대신 외부 브라우저가 열린다
정상 폴백 동작(pywebview 두 번째 창 생성 실패). 로그인 자체는 된다.

### 로그인 창이 아예 안 뜬다
로그인 버튼 아래 **"인증 코드로 로그인"** 을 쓰면 된다(리디렉션 URI 불필요).

### 다른 사람이 로그인 못 함
2단계의 "지원되는 계정 유형" 확인. "개인 Microsoft 계정" 이어야 한다.

---

## 참고: Client ID 노출에 대해

Client ID 는 exe 에 포함되어 배포되지만 **비밀값이 아니다**.
- client secret 이 없는 퍼블릭 클라이언트라 Client ID 만으로 남의 계정에 접근 불가
- 최악의 경우 누가 이 ID 로 자기 앱을 만들어 요청 한도를 쓰는 정도
- 커뮤니티 런처 규모에서는 현실적 위협이 아니다
